Documentation simplified search

This documentation outlines the functions provided by the simplified search.

Below the single parts of the simplified search are listed and illustraited.

Simplified search

The simplified search page consists of three parts. The upper part merely states the project name, but also offers a link to the simplified search, the ANNIS search tool, and this documentation page.

The middle part consists of a large query box in which a query can be entered. More information about this box is provided below under "Query box" and "Search area".

At the bottom of the page, tickboxes allow the user to constrain the query in the query box to a number of categories. More information on these meta-categories is provided below under "Meta constraints".

Query box

The query box is set up to parse the input that it receives in such a way that a more complex ANNIS query can be written. You may enter a single word or multiple word queries.

The parser of the query box also automatically resolves diacritics to the form that occurs in the corpora.

Finally, it is possible to use simple regular expressions in the query box.

Search area

The search area specifies how your query is interpreted.

You can choose between a word form search and a lemma search. The word form is the form the word occurs in the text. As word forms there are used simplified forms generated from the transcription.
A lemma is a generalized word form. For example a lemma entry in REM is bruoder (brother). The lemma bruoder can be realized as different word forms, e.g. identical as bruoder, but also as bruder, brvder, pruder or even brudir. Of course the word forms highly depend on the time of their text creation and dialect areas. But it is always the same lemma bruoder represented.

If you are not sure which word forms could possibly occur in the texts or if you do not want to restrict your query on concrete word forms, you should use the lemma search. Of course you still have to know the exact Middle High German lemma entries. A good start for getting these lemma entries is the Middle High German lexicon of Matthias Lexer (in German).

Furthermore you have the option to let your query be part of another word. Choose "Whole word" to simply search your query as typed in. For example if you enter mit in the query box by choosed "Whole word", the string mit is searched and all occurrences of mit are found. If you enter mit into the query box and choose "Word starts with query" (and a word form search), the search finds mit, but also e.g. mite (a word form of the lemma mit(e)), or mitte and mitten (word forms of the lemma mitte). Choose "Word ends on query" to find e.g. goumit (lemma: goumen) or urumit (lemma: vrumen). In all cases also mit will be found.

Meta constraints

The meta categories provide constraints on the text search:

  • "Dating" is based on the century in which the text is supposedly written down
  • "Text field" specifies a certain topic the text can be associated with
  • "Dialect area" is based on an interpretation of the dialect area that some texts may be typical for
The dialect areas can be splitted into the following dialects:
  • West Middle German: Middle Franconian, Rhenish Franconian, Moselle Franconian, Ripuarian, Hessian
  • East Middle German: South Thuringian, Thuringian, Upper Saxonian
  • West Upper German: Alemannic, Swabian, Alsatian
  • East Upper German: Bavarian, Austrian
  • North Upper German: East Franconian, Rhenish Franconian, South Rhenish Franconian, Nurembergian
If you want to choose a single dialect, you should formulate or modify a query using the advanced search. Please see the section "For advanced searching" for further information.

If no constraint is set, all texts are searched. Every checked tickbox constrains the searched texts. If you check "12th century", "14th century", "religion" and "East Middle German", only East Middle German texts dating from the 12th and 14th century with religion as their topic are searched through your query.

For advanced searching

If you want to formulate more complex and specific queries, you should use the ANNIS search tool directly. This simple search redirects to the ANNIS search tool, but secretly formulates a formal query ANNIS can work up (you can see this query in ANNIS on the left side, if you ran a query with the simplified search).

Every user can formulate such a formal query and let ANNIS search with it. The used formal language is called AQL (ANNIS Query Language). For further information on AQL and its structure see http://annis-tools.org/aql.html.

Example queries

A number of things that can be typed in for your inspiration:

  • jesus christus
  • got, as a word form search and constraints on the 13th century
  • maria as a lemma search and constraints on West Middle German
  • maria as a word form search
    (Notice the different realizations of maria in the different texts)
  • umbe as lemma search and with checked "Word ends on query"
    (this way the search also finds compounded lemma entries such as wâr/+umbe and dâr/+umbe. As word forms this would be seperated words: wâr umbe and dâr umbe)
  • h[vu]nd[ei]rt as word form search
    (This is a regular expression. The square brackets stand for an option. It is searched for hvndert, hundert, hvndirt and hundirt)
  • ...
